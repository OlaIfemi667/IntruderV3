What is Intruder?

Automating the footprinting phase of penetration testing

Intruder is a tool written in Python that automates the early phases of a penetration test, such as reconnaissance and scanning.

It integrates tools like Nmap, Exploit-DB, WHOIS, and OWASP ZAP to simplify tasks, and leverages an LLM for intelligent analysis and reporting.

Key Penetration Testing Phases

1. Footprinting & Reconnaissance

  • Network enumeration
  • Domain information gathering
  • WHOIS lookups
  • DNS enumeration
  • Port scanning
  • Service identification

2. Scanning

  • Open ports and services
  • Operating system detection
  • Vulnerability scanning
  • Network mapping

3. Enumeration

  • User accounts
  • Machine names
  • Network resources
  • Services and applications

How does it work?

Intruder acts as a wrapper around essential security tools:

  • Nmap for scanning ports and services
  • WHOIS for domain registration data
  • Exploit-DB for known vulnerabilities
  • OWASP ZAP for web application analysis

Results from these tools are interpreted using a Large Language Model to assist the user in report generation and understanding the risks.
